# Пошаговый план разработки (Prompt Plan)

Разработка архитектуры и контента велась строго по этапам на основе документа 01-landing-client-content (1).docx.

Шаг 1: Анализ контента и разметка структуры (HTML)
- Проведен анализ сырого клиентского текста. Выстроена логическая иерархия из 8 блоков.
- Написана семантическая разметка index.html с использованием тегов header, main, section, form, footer.
- Интегрированы текстовые блоки: спецификации калибров, этапы операционного цикла и болгарские юридические реквизиты компании.

Шаг 2: Визуальное оформление (CSS)
- Реализован современный дизайн с темным фоном и карточками в стиле «матового стекла» (эффект backdrop-filter).
- Оформлены списки требований сетей (Блок 5) и форматы фасовки сырья с кастомными маркерами.
- Добавлена адаптивность для корректного отображения структуры на мобильных устройствах.

Шаг 3: Бизнес-логика и интерфейсные сценарии (JS)
- Создан изолированный файл script.js для разгрузки HTML-страницы.
- Реализован перехват события submit формы обратной связи для выполнения жестких условий валидации ТЗ (запрет отправки без чекбокса, проверка регулярным выражением для e-mail).
- Добавлен обработчик клика на CTA-кнопку главного экрана для плавного скролла к форме.

Шаг 4: Развертывание юридического слоя (GDPR)
- Тексты политик из финальной части клиентского документа вынесены в независимые файлы privacy.html и cookies.html.
- Настроена корректная переадресация из футера главного сайта на страницы документов.
