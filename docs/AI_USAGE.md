# Отчет об использовании AI инструментов (AI Usage Report)

В процессе проектирования и верстки B2B-платформы NUTRI TRADE LTD я использовала искусственный интеллект в качестве инжинирингового ассистента под моим личным управлением. Ответственность за архитектуру, финальную сборку и проверку работоспособности полностью лежала на мне.

Распределение задач в ходе работы над проектом:

## 1. Архитектура и структура сайта
- Я самостоятельно спроектировала каркас одностраничного сайта и логику его разделения на дополнительные разделы.
- Мной была написана семантическая разметка `index.html` и настроена базовая структура хедера, секций и футера на основе исходного текстового файла заказчика.

## 2. Структурирование контента и юридических блоков (Совместное выполнение)
- Из сырого клиентского файла я выделила ключевые коммерческие блоки для наполнения лендинга.
- С помощью ИИ-ассистента я адаптировала и развернула краткие тезисы из ТЗ в полноценные, юридически грамотные тексты для *Политики конфиденциальности* (Privacy Policy) и *Политики Cookies*, соответствующие правилам GDPR. 
- Юридические тексты были упакованы в интерактивные модальные окна внутри `index.html` для экономии экранного пространства без потери важных смыслов.

## 3. Интерактив, валидация и UI-компоненты (Совместное выполнение)
- **Валидация формы:** Я сформулировала требования к проверке обязательных полей формы и e-mail без перезагрузки страницы. ИИ-ассистент помог сгенерировать базовый JS-код для валидации, регулярное выражение для проверки почты и логику переключения формы в состояние `success` (сообщение об успешной отправке).
- **Оптимизация интерфейса (Аккордеон):** Чтобы не превращать страницу в длинный текстовый документ, мы совместно с ИИ разработали интерактивный аккордеон для секции «Как мы работаем», перенеся туда весь массивный операционный цикл без сокращений.
- **Cookie-баннер:** ИИ помог реализовать всплывающее уведомление (баннер) с интеграцией с `localStorage` браузера, чтобы окно автоматически скрывалось после клика на кнопку «Принять» и не беспокоило пользователя при повторных визитах.

## 4. Поиск ошибок и отладка (Дебаг)
- Я привлекала ИИ для локализации точечных багов в коде: с его помощью был найден скрытый мусорный символ в начале HTML-документа, ломавший разметку, решена проблема с зависанием портов в расширении Live Server, а также оперативно локализован и устранен конфликт путей из-за дублирования файлов `index.html` в структуре проекта.

---

## Итог
Инструменты ИИ были использованы мной для автоматизации рутинных задач (генерация шаблонов документов, базовых регулярных выражений, обработчиков кликов для UI-элементов). Это позволило мне сосредоточиться на UX-логике, верстке интерфейса, адаптивности и жестком контроле соответствия проекта исходному ТЗ заказчика.
